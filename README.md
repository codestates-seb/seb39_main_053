## 🔀 Git Flow Branch Strategy
- `main` : 제품으로 출시할 수 있는 브랜치
- `dev` : 출시 버전을 개발하는 브랜치
- `feat` : 기능을 개발하는 브랜치

## 📌Git Commit Convention

```
type(태그에서 선택): Subject  // -> 제목
(empty line)
body(옵션)  // -> 본문 
```

|태그 이름|설명|
|--|--|
|**Feat**|새로운 기능 추가|
|**Fix**|버그 수정|
|**Design**|UI 디자인 변경|
|**Refactor**|코드 리팩토링|
|**Docs**|문서 수정|
|**Test**|테스트 코드 작성|
|**Rename**|파일 명 또는 폴더 명 수정|
|**Remove**|파일 삭제|
